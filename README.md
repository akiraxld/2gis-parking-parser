# 2ГИС Парсер парковок Алматы

Сборщик данных о парковках города Алматы из 2ГИС. Собирает структурированные данные и выгружает в Google Sheets.

📊 **[Результат: Google Sheets с данными](https://docs.google.com/spreadsheets/d/1Hjb0XcDKIqNgTikKE9Ok74ev0xFajF0QiPdhiavlNzQ/edit?gid=2013927562#gid=2013927562)**

---

## Что собирает

956 уникальных парковок Алматы со следующими полями:

| Поле | Заполнено | Описание |
|------|-----------|----------|
| Название | 100% | Название объекта |
| Район | 99% | Административный район |
| Платная | 99% | Да / Нет |
| Вместимость | 97% | Количество мест |
| Тариф | 93% | Текст тарифа как в 2ГИС |
| Цена/час | 90% | Числовое значение в тенге |

| Тип | 100% | ТЦ / БЦ / ЖК / городская / паркинг |

| Часы работы | 100% | Расписание в читаемом формате |

## Архитектура

Проект работает в два этапа.

**Этап 1 — Сбор ID через Playwright**

Публичный ключ 2ГИС блокирует прямые запросы из Python. Поэтому используется Playwright: браузер делает запросы к `/3.0/markers/clustered` сам, скрипт перехватывает ответы через `page.on("response", ...)`. Алматы разбивается на сетку 12×12 = 144 ячейки, для каждой ячейки браузер перемещается в её центр и перехватываются объекты в этом вьюпорте. Результат — 956 уникальных `id_2gis` в SQLite.

**Этап 2 — Обогащение через официальный API**

Официальный ключ с [developer.2gis.ru](https://developer.2gis.ru) работает из Python напрямую. Каждый собранный ID прогоняется через `/3.0/items/byid` батчами по 50 штук — получаем полные карточки с адресом, тарифом, вместимостью, расписанием и атрибутами. Итог сохраняется в `raw_items.rich_json`.

```
Playwright (браузер)          Официальный API
  сетка 12×12 ячеек    →    /3.0/items/byid
  /3.0/markers/clustered      батчи по 50 ID
  956 уникальных ID     →    полные карточки
        ↓                          ↓
              SQLite (parkings.db)
                      ↓
              Google Sheets (956 строк)
```

## Быстрый старт

**1. Клонировать и установить зависимости**

```bash
git clone https://github.com/akiraxld/2gis-parking-parser.git
cd 2gis-parking-parser
pip install -r requirements.txt
playwright install chromium
```

**2. Создать `.env` файл**

```bash
# .env
DGIS_API_KEY=ваш_ключ_с_developer.2gis.ru
```

Бесплатный ключ: [developer.2gis.ru](https://developer.2gis.ru) → создать проект → API Keys.

**3. Настроить Google Sheets** (для выгрузки)

- Создать проект в [Google Cloud Console](https://console.cloud.google.com)
- Включить Google Sheets API и Google Drive API
- Создать сервисный аккаунт → скачать JSON → сохранить как `service_account.json`
- Поделиться таблицей с email сервисного аккаунта (права редактора)

**4. Запустить сбор**

```bash
# Полный сбор: Playwright → SQLite → обогащение через API
python collector.py

# Только обогащение (если ID уже собраны)
python collector.py --enrich-only

# Выгрузка в Google Sheets
python export.py
```

## Технические решения и их обоснование

**Почему Playwright, а не прямые запросы для сбора ID**

Публичный ключ 2ГИС (`c7f1a769...`) встроен в сайт и общий для всех пользователей. При прямых запросах из Python он немедленно блокируется (`apiKeyIsBlocked`). Playwright запускает настоящий браузер — запросы идут с правильными куками и заголовками, ключ работает.

**Почему сетка координат, а не один поиск**

Endpoint `/3.0/markers/clustered` отдаёт максимум ~103 объекта на один вьюпорт. При сетке 12×12 каждая ячейка покрывает небольшой район — объектов меньше, кластеризация минимальна. Дедупликация по `id_2gis` устраняет дубли из перекрывающихся ячеек.

**Почему официальный API для обогащения**

После сбора ID нужны полные карточки объектов. Endpoint `/3.0/items/byid` принимает до 50 ID за раз и возвращает все поля. Официальный ключ не требует сессионных параметров и не блокируется — 956 объектов обогащаются за ~20 секунд.

**Почему SQLite, а не файлы**

Сбор прерывается и возобновляется (`--resume`). SQLite хранит статус каждой ячейки (`pending / done / failed`) и гарантирует что при перезапуске уже обработанные ячейки пропускаются. `INSERT OR IGNORE` по `id_2gis` исключает дубли автоматически.

## Ограничения

- **Адрес** заполнен только у 4% объектов — 2ГИС не хранит точные адреса для большинства парковок, только координаты.
- **EV-зарядка и охраняемость** — эти атрибуты крайне редко заполнены в самом 2ГИС.
- **~134 парковки из ~1090** не попали в выборку — они находятся за пределами bbox Алматы или помечены другой рубрикой в 2ГИС.
